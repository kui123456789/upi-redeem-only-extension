# UPI Redeem Only 配置使用说明

本文档只说明当前 UPI 专用版本的配置和常用操作。

## 1. 安装加载

1. 打开 Chrome：`chrome://extensions/`
2. 开启右上角“开发者模式”。
3. 点击“加载已解压的扩展程序”。
4. 选择本项目目录：`upi-redeem-only-extension`。
5. 如果要在无痕窗口跑流程，进入扩展详情页，开启“在无痕模式下启用”。

更新代码后，需要在 `chrome://extensions/` 里点击该扩展的“重新加载”，再重新打开侧边栏。

## 2. 必填配置

UPI 专用版已经固定使用下面两个远端服务：

- 兑换后端：`https://chong.nerver.cc/api/external/cdkey-redeems`
- 状态/资格/会员查询：`https://cha.nerver.cc`

侧边栏里通常只需要配置：

1. `UPI Key`
   - 填后端提供的 `X-External-Api-Key`。
   - 只用于提交兑换和刷新卡密远端状态。
   - 不要填写 `Bearer` 前缀，只填 key 本身。

2. `UPI 卡密池`
   - 一行一个卡密。
   - 示例：

```text
CDK-001
CDK-002
CDK-003
```

3. `UPI Client ID`
   - 可留空。
   - 留空时扩展会自动生成并保存在本地设置中。

4. `订阅 API`
   - 默认是 `https://cha.nerver.cc`。
   - 没有特殊要求不要改。

## 3. 账号导入格式

Free 分组和备份账号导入使用同一种 TXT 格式：

```text
邮箱---GPT密码---2FA密钥
```

示例：

```text
demo1@icloud.com---Password123!@#---JBSWY3DPEHPK3PXP
demo2@icloud.com---Password456!@#---MFRGGZDFMZTWQ2LK
```

规则：

- 每行一个账号。
- 空行和 `#` 开头的行会跳过。
- 导入时按邮箱去重；同一个邮箱只保留一条。
- 账号必须有 GPT 密码和 2FA 密钥，才能自动重新登录、检测会员、提交卡密。

## 4. 常用按钮

UPI 卡密池区域常用按钮：

- `刷新状态`
  - 查询后端卡密兑换状态。
  - UPI 面板可见且已配置 key/卡密时，会每 5 秒自动刷新。

- `导入无会员TXT`
  - 把 TXT 账号导入 Free 分组。
  - 导入后点击 Free 分组账号，可重新登录账号并提交卡密兑换。

- `导入备份TXT并核验`
  - 导入账号后先检测是否已有 Plus。
  - 有会员进入 Plus 分组，无会员进入 Free 分组。

- `核验启用已存备份`
  - 对本地已保存的账号重新核验会员状态。

- `导出已开通会员密码2FA`
  - 导出 Plus 分组账号。
  - 导出当前批次后会清理对应分组，避免下一批混在一起。

- `查看全部已存密码2FA`
  - 查看本地保存的账号备份。

## 5. 推荐流程

### 新注册账号自动兑换

1. 配好 `UPI Key`。
2. 导入 `UPI 卡密池`。
3. 开启 UPI 流程运行注册。
4. 程序到第 7 步后会记录账号，并检测是否有试用资格。
5. 有试用资格的账号进入 Free 分组。
6. 第 8 步读取当前 ChatGPT session，并提交 `session + access_token + 卡密` 到兑换后端。
7. 后端返回成功后，卡密状态显示 `兑换成功`，对应账号进入 Plus 分组。

### Free 分组补兑

1. 点击 `导入无会员TXT`，导入格式为 `邮箱---GPT密码---2FA密钥`。
2. 点击 Free 分组里的账号，程序会重新登录该账号。
3. 读取 ChatGPT session。
4. 提交当前可用卡密到兑换后端。
5. 提交成功后等待远端状态刷新，不立即删除卡密。

### Plus 分组核验

点击 Plus 分组账号会检测当前账号是否仍有会员：

- 有会员：保留在 Plus 分组。
- 无会员：自动回到 Free 分组。

## 6. 卡密状态规则

卡密状态以远端返回为准：

- `兑换成功`
  - 后端返回 `success`。
  - 该卡密不会再次参与兑换。

- `处理中`
  - 后端返回 `submitted`、`running`、`processing`、`pending_dispatch` 等。
  - 等待自动刷新，不要手动重复提交同一账号。

- `失败`
  - 后端返回 `failed`、`timeout`、`rejected`、`canceled`、`not_found` 等。
  - 失败原因会显示在状态说明里。

- `邮箱不可用/approve-blocked`
  - 程序会释放卡密，并删除或标记该账号不可再用。

重要规则：

- 卡密本身不做自动重试。
- 卡密可以继续保留在池里，是否可用由远端状态决定。
- 失败三次自动删除的是账号，不是卡密。
- 卡密不足时会停止，不会自动跳到下一个账号继续乱跑。

## 7. 试用资格检测

资格检测使用：

```text
POST https://cha.nerver.cc/api/v1/check
```

请求只需要 ChatGPT access token，不需要 UPI Key，也不需要卡密。

检测结果：

- 有试用资格：账号保留在 Free 分组，可继续兑换。
- 明确无资格：账号自动删除或标记已用。
- token/session 过期：不会直接判定账号无资格，会尝试刷新当前 ChatGPT 页面后继续。

## 8. 常见错误

### `UPI 远端接口认证失败 / HTTP 401 / HTTP 403`

检查 `UPI Key` 是否正确、是否有兑换接口权限。

### `Invalid CORS request`

通常是后端不允许当前请求来源，或后端网关配置拒绝了扩展请求。保留日志里的完整后端返回信息，按后端配置排查。

### `access_token 不能为空`

表示提交到兑换后端的 item 缺少 `access_token`。当前版本会从 ChatGPT session 中提取并随 `session` 一起提交；遇到这个错误请先确认已经重新加载最新扩展。

### `ChatGPT session 已过期或当前会话失效`

当前 ChatGPT 标签页登录态失效，或读取到的 session 不完整。重新登录该账号后再重试。

### `账号登录态不一致`

程序检测到当前已登录账号和目标邮箱不同。应停止当前补兑，重新登录目标邮箱后再提交，避免卡密提交到错误账号。

### `UPI 卡密不足`

当前可用卡密数量不足。程序会停在当前账号，不会继续处理下一个账号。补充卡密后再继续。

## 9. 导出和批次管理

Plus 分组导出格式：

```text
邮箱---GPT密码---2FA密钥
```

导出 Plus 分组时会清理当前导出的分组结果，下一批账号不会和这一批混在一起。

建议每批兑换完成后：

1. 点击 `刷新状态`，确认卡密远端状态。
2. 导出 Plus 分组。
3. 保存导出的 TXT。
4. 再开始下一批账号。
